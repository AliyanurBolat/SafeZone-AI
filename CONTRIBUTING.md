# Contributing to SafeZone AI 🤝
First of all, thank you for your interest in contributing to SafeZone AI! Projects like this, aligned with Digital Public Goods (DPG) principles, rely on community collaboration to improve workplace safety worldwide.
By contributing, you help us achieve SDG 8: Decent Work and Economic Growth by making industrial environments safer through open-source innovation.
## 🏗️ How You Can Contribute
We welcome contributions of all kinds, regardless of your technical level:
 1. Reporting Bugs: If you find a bug in our detection logic or documentation, please open an Issue on GitHub.
 2. Feature Requests: Have an idea for a new safety feature (e.g., hard hat detection or fire alerts)? Open an issue to discuss it.
 3. Code Improvements: Help us optimize our Python scripts or improve our computer vision algorithms.
 4. Documentation: Clear instructions are vital for DPG. Helping us improve our README.md or adding tutorials is highly valued.
## 🛣️ Our Contribution Process
To ensure a smooth workflow and high-quality code, please follow these steps:
### 1. Fork the Repository
Create your own copy of the project by clicking the Fork button at the top of the page.
### 2. Create a Branch
Work on your changes in a new branch to keep the main code stable:
git checkout -b feature/your-awesome-feature### 3. Commit Your Changes
Use clear and descriptive commit messages (e.g., feat: add helmet detection logic).
### 4. Push and Open a Pull Request (PR)
Push your branch to your fork and submit a Pull Request. A team member will review your contribution as soon as possible.
## 📜 Coding Standards
 * Language: Please write all code, comments, and documentation in English to ensure global accessibility.
 * Documentation: Every new function should include a brief docstring explaining its purpose.
 * Dependencies: If you add new libraries, remember to update the requirements.txt file.
## ⚖️ Open Source Ethics
 * License: By contributing to SafeZone AI, you agree that your contributions will be licensed under the MIT License.
 * DPG Alignment: Ensure that your contributions do not violate privacy standards and adhere to the "Do No Harm" principle of Digital Public Goods.
## 👥 Need Help?
If you have questions about the project or the contribution process, feel free to reach out to the project maintainers listed in the README.md.
*Happy coding for a safer world!* 🛡️
